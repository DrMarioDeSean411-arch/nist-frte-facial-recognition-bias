Per-cell false match rates (FMR) across five demographic regions and five age groups. Color encodes magnitude — darker red means more false matches. Select an algorithm and sex to update the matrix. Click any cell for an operational interpretation.

What FMR means operationally: A false match rate of 0.03 means that for every comparison of two photos of different people, the algorithm incorrectly declares them the same person 3% of the time. In a 1:N search against a database of 10,000 people, this translates to ~300 false investigative leads for a single query. The same algorithm produces near-zero false matches for Eastern European males aged 35–50 — the demographic group that consistently performs best across 95%+ of NIST-evaluated algorithms (NIST, 2026; Grother, 2022).

Methodology: FMR values are extracted from NIST FRTE published log₁₀ FMR heatmaps (Figure 1 and Annex A, NISTIR 8429). Threshold set per algorithm to achieve nominal FMR = 0.00001 overall on the visa-border dataset. Values ≤ 10⁻⁶ are reported as < 10⁻⁶ (displayed as < 0.000001). Race is proxied by country of birth per NISTIR 8429 §4.1. Within-region FMR is computed by pooling intra- and inter-country comparisons within each region (eq. 25, NISTIR 8429). FMR values for qazsmartvisionai_003 and innovatrics_014 are interpolated from published NIST FRTE demographic tables; exact cell-level log₁₀ FMR for all algorithms is available at the NIST FRTE results page linked above.
